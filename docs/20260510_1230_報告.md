## 2. 決定済み事項・要件

### 基本ルール

ステージ内に動的に生成されるコインを全回収するとステージクリア。

### 難易度設計

ステージ進行に応じてコイン生成数を増加させる。

### ミス判定

プレイヤーのY座標が `1000` を超えた場合、落下ミスとして扱う。

### 画面仕様

以下の解像度を切り替え可能とする。

- 1152x648
- 1920x1080

### 公開予定

GitHub Pagesを利用したWeb公開を目指す。

---

## 3. GitHub運用ルール

### ブランチ戦略

役割ごとに以下の5分類で管理する。

| 種類 | 用途 |
|---|---|
| Release | リリース管理 |
| Feature | 新機能開発 |
| Develop | 開発用メインブランチ |
| Fix | バグ修正 |
| Other | その他作業 |

### 命名規則

```text
develop/v-X.X.X/機能名

例：
develop/v-0.1.0/player_base
```
### 運用方針
1機能につき1ブランチ
1作業単位ごとにコミット
Pull Requestによるレビューを実施
マージ前に必ず動作確認を行う
